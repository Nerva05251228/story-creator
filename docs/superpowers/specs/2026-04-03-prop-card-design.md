# 道具卡完整接入设计

日期：2026-04-03

## 目标

在现有主体体系中新增 `道具` 卡片类型，并将其作为和 `角色`、`场景` 同级的主体类型完整接入全链路。

接入后的主体类型分层如下：

- 主体卡：`角色` / `场景` / `道具`
- 附属卡：`声音`

本次变更要求 `道具` 不只是主体页多一栏，而是进入主体抽取、AI 提示词生成、镜头主体选择、故事板 2、镜头图/视频参考图携带等核心流程。

## 非目标

- 不给道具增加“三视图”能力。
- 不允许声音卡绑定道具或场景，仍然只绑定角色。
- 不让道具参与场景描述抽取或镜头场景图替换逻辑。
- 不修改现有声音卡自动补齐机制，只保持其绑定角色的现状。

## 用户可见行为

### 主体页

主体页从三栏变为四栏：

- 角色
- 场景
- 道具
- 声音

道具卡支持：

- 手动新建
- 编辑名称、alias、ai_prompt
- 生成 AI 提示词
- 普通作图
- 上传图片
- 选择主体素材图
- 在镜头中被选中作为主体参考

道具卡不支持：

- 主角设置
- 三视图
- 声音绑定

### 镜头与故事板 2

道具卡会像角色卡一样进入主体选择区域，并可被选入：

- 分镜表镜头主体
- 故事板 2 镜头主体
- 故事板 2 子镜头主体

参考图收集时，道具会和角色一起参与“主体参考图”携带；场景是否携带仍由现有场景开关控制。

## 数据与类型设计

### 类型常量

将主体类型常量调整为：

- `ALLOWED_CARD_TYPES = ("角色", "场景", "道具")`
- `ALL_SUBJECT_CARD_TYPES = ("角色", "场景", "道具", "声音")`
- `SOUND_CARD_TYPE = "声音"`

语义：

- `ALLOWED_CARD_TYPES` 表示主主体链路允许的类型，用于主体抽取、AI 提示词生成、镜头主体选择、参考图携带。
- `ALL_SUBJECT_CARD_TYPES` 表示主体库允许创建和编辑的全部卡片类型。

### 字段复用

不新增表字段。`SubjectCard` 继续复用现有字段：

- `name`
- `alias`
- `card_type`
- `ai_prompt`
- `role_personality`
- `style_template_id`

其中：

- 角色：`role_personality` 有意义
- 场景 / 道具：`role_personality` 始终写空字符串
- 声音：保留当前逻辑

## AI 与 Prompt 设计

### Stage1 / 详细分镜主体抽取

所有“主体类型只有角色 / 场景”“不输出道具”的提示词改为支持 `角色 / 场景 / 道具`。

主体提取规则调整为：

- 每个镜头可包含 `1-2` 个主要角色
- 必须包含 `1` 个场景
- 可包含 `0-2` 个关键道具
- 只保留推动画面识别和镜头生成所必需的道具，不把零碎杂物都提成主体

### Stage2 主体去重与提示词生成

`stage2_refine_shot` 改为支持三类主体：

- 角色：现有规则不变
- 场景：现有规则不变
- 道具：增加单独的提示词规则

道具 `ai_prompt` 结构统一为：

- 材质
- 造型
- 颜色
- 结构特征
- 使用痕迹 / 新旧程度
- 关键细节

道具 `alias` 仍然是 10-20 字简短描述。

`role_personality`：

- 角色填写中文一句话性格
- 场景 / 道具必须为空字符串

### 单卡 AI 提示词生成

`generate_subject_ai_prompt` 也扩展为支持 `道具` 类型，规则与 Stage2 保持一致，避免批量生成和单卡生成行为分叉。

### PromptConfig 升级策略

当前项目大量 prompt 模板存在数据库 `prompt_configs` 中，因此不能只改代码默认值。

处理策略：

- 升级内置默认模板文案，新增“道具”说明
- 对已存在数据库配置做“安全升级”
- 仅在内容明显仍是旧版内置模板时自动替换
- 用户已自定义明显不同的模板内容不强行覆盖

这保证：

- 新环境有正确默认值
- 旧环境能自动补到支持道具
- 管理端自定义配置尽量不被破坏

## 后端改动范围

### 主体标准化与落库

以下链路统一支持 `道具`：

- `_normalize_subject_detail_entry`
- `_normalize_stage2_subjects`
- 批量生成主体提示词
- 主体库卡片创建 / 更新 / 查询
- 基于 stage2 结果创建主体卡

### 镜头主体查询与排序

当前多处默认排序是“角色优先，场景次之”。接入道具后统一为：

- 角色
- 场景
- 道具

声音卡继续走独立链路，不混入主体主排序。

### 参考图携带

所有基于 `ALLOWED_CARD_TYPES` 收集主体图的地方都会自动带上道具卡。

对场景仍保留特殊判断：

- 场景是否携带参考图继续受现有开关控制
- 道具不走场景特例，默认和角色一起进入主体参考链路

### 场景描述抽取

场景描述抽取仍然只看 `场景` 卡，不看 `道具`。

需要保证以下逻辑不被道具污染：

- `extract_scene_description`
- `scene_override` 自动填充
- 镜头上传场景图替换

## 前端改动范围

### 主体页

主体页新增“道具”栏：

- 新建道具按钮
- 独立容器渲染
- 空状态文案

侧边栏编辑逻辑增加道具分支：

- placeholder 改为道具文案
- 显示普通 AI 提示词编辑能力
- 不显示角色性格输入框
- 不显示三视图按钮

### 主体卡渲染

渲染逻辑按类型分流：

- 角色：现状保持
- 场景：现状保持
- 道具：走图片主体卡逻辑
- 声音：现状保持

### 镜头主体选择

以下界面新增“道具”分组：

- 故事板主体选择区
- 故事板 2 镜头主体选择区
- 故事板 2 子镜头主体选择区

分组顺序保持为：

- 角色
- 场景
- 道具
- 声音

其中声音继续单独控制 `selected_sound_card_ids`；道具属于普通 `selected_card_ids`。

### 默认命名与表单选项

所有前端硬编码的主体类型列表同步补齐：

- 新建卡片默认名
- 下拉选项
- 类型标签
- 过滤条件

## 兼容性与迁移

### 旧数据兼容

现有 `SubjectCard` 表不需要迁移结构，只需要允许新的 `card_type='道具'` 值进入业务逻辑。

### 旧 prompt 兼容

旧环境若 prompt 配置仍然写着“只允许角色/场景”，会导致 AI 永远不产出道具，因此必须做配置升级。

### 旧镜头兼容

老镜头没有道具时不需要额外处理。
接入后只有新生成或新编辑的镜头才会逐步带上道具主体。

## 风险点

### 风险 1：AI 过度抽取道具

如果 prompt 改得太宽，AI 可能把无关杂物都提成主体。

缓解：

- 在 prompt 中明确“只保留关键道具”
- 限制每镜头 `0-2` 个道具

### 风险 2：场景逻辑被道具污染

当前有不少逻辑默认“非角色即场景”，这会让道具误入场景链路。

缓解：

- 所有场景专属逻辑改成显式判断 `card_type == "场景"`
- 不再依赖“else 就是场景”

### 风险 3：前端选择器漏改

前端很多地方写死了角色/场景/声音三组，容易出现主体页改了但镜头选择没改。

缓解：

- 统一梳理所有 `card_type ===` 和主体分组代码路径
- 对故事板、故事板 2、主体页分别做回归测试

## 测试策略

### 后端

新增或更新测试覆盖：

- 类型常量允许 `道具`
- stage2 主体标准化保留 `道具`
- 单卡 AI prompt 生成输入允许 `道具`
- 参考图收集会带上道具
- 场景描述抽取不会错误读取道具

### 前端

新增或更新测试覆盖：

- 主体页四栏渲染包含道具
- 道具卡不显示三视图
- 镜头主体选择区包含道具分组
- 道具进入 `selected_card_ids`
- 声音仍然只进 `selected_sound_card_ids`

### 人工验证

至少验证以下流程：

1. 主体页新建道具卡
2. 为道具卡生成 AI 提示词
3. 为道具卡作图并设为主体素材图
4. 在镜头主体里选择道具
5. 生成故事板 2，确认道具能出现在主体列表
6. 生成镜头图或视频，确认道具参考图被携带
7. 确认场景描述仍只来自场景卡
